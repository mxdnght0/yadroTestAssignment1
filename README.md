# DNS Manager

Выполнил задание по разработке клиент-серверного приложения для управления DNS-серверами на Go.

Сервер предоставляет REST API на базе Gin, который позволяет добавлять, удалять и получать список DNS-серверов. Все записи хранятся в файле `/etc/resolv.conf` в формате `nameserver <ip>`. Клиент реализован как CLI-приложение на Cobra с командами `add`, `delete`, `list` и поддержкой `--help`.

Для отладки сервер пишет структурированные JSON-логи через `slog` с уровнями `INFO`, `WARN`, `ERROR`. Уровень логирования переключается через переменную окружения `LOG_LEVEL=debug`.

Для запуска сервера нужно задать переменные окружения:

```
RESOLV_PATH=/etc/resolv.conf   # путь к файлу DNS-серверов
SERVER_PORT=8000                # порт сервера
LOG_LEVEL=info                  # уровень логирования (info / debug)
```

Для запуска CLI адрес сервера задаётся через:

```
DNS_SERVER_HOST=http://localhost:8000
```

Тесты запускаются командой:

```
go test ./server/... -v
```

На каждый push и pull request настроен CI через GitHub Actions, который автоматически запускает тесты.
